Demo Exceptions

Partie 1

Dans cette partie, nous allons manipuler les exceptions : les buts sont de savoir comment gérer les blocs try-catch, gérer les Exceptions en chaîne, et créer sa propre Exception.

Nous utilisons le package exceptions de l’archive importée.

A.

Compléter la classe Exception2 en attrapant le plus possible d’exceptions. Créer un array de Strings testArgs contenant les quatre éléments suivants : "1", "0", "Except", et "Test".

  1. Pour l’instant ne pas toucher aux méthodes effetDomino 1, 2, 3 et lanceUneException().
  2. Dans le main:
  • Appeler la méthode StringToInt avec testArgs[0]
  • Appeler la méthode getInverse avec Integer.parseInt(testArgs[1])
  • Créer un objet de type Exception2 (avec un String testArgs[2] en paramètre) et appeler getLength
  • Apppeler la méthode effectuerChangements avec comme paramètre un nouvel objet de type Exception1

B.

Créer la classe DivideByZero qui étend la classe ArithmeticException. Cette classe définit l’exception qui consiste à diviser par zéro.

  • Créer son constructeur
  • Créer sa méthode message() qui affiche "Vous êtes bien dans la classe DivideByZero"

C.

Modifier la méthode lanceUneException pour qu’elle lance une exception de type DivideByZero. Les méthodes effetDomino 1 et 2 tenteront de l’attraper et d’appeler la méthode message().

D.

Tenter d’accéder à testArgs[3]

E.

Ouvrir la classe Exception1. Que se passe-t-il si :

  • dummy() lance une NullPointerException ?
  • dummy() lance une ArithmeticException ?
  • dummy() lance une IOException ?
  • dummy() lance une ArrayIndexOutOfBoundException ?
  • dummy() réussit à aller au bout ?

Faire les tests nécessaires.

Partie 2

Dans cette partie, nous nous intéressons à un outil prenant en entrée des notes manuscrites (donc écrites sous forme de String), et nous allons essayer de traiter les types d’erreur existants pouvant apparaître pendant l’utilisation de cet outil.

A.

Ecrivez dans une classe Average une méthode fromStringArray() qui demande à l’utilisateur d’entrer une liste de notes, puis calcule et retourne la moyenne des notes entrées.

Vous utiliserez la méthode Integer.parseInt qui transforme une chaine de caractères en entier et lève une exception NumberFormatException si la chaîne n’est pas un entier.

B.

Que se passe-t-il lorsque l’on exécute le programme avec la liste "1" "2" "30" "1111" ?

Avec la liste "1" "2" "30" "1111K" ?

Avec une liste vide ?

C.

On souhaite maintenant que lorsqu’une note non entière est entrée, le programme ignore cette note et affiche le message “erreur la note [NOTE] n'est pas entière", où [NOTE] est la note qui pose problème. De plus si aucune note valide n’est entrée, le message "Il faut au moins une note" s’affiche.

D.

Écrivez une classe NoteNonConformeException dérivée de la classe Exception et qui affiche "NoteNonConformeException: La note [NOTE] est non conforme (une note entre 0 et 20 est attendue)" où [NOTE] est la note problématique.
